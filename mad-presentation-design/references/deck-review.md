# Deck review

Используйте этот справочник для deck review и финальной полировки.

## Критерии

### 1. Ясность сообщения

- Главная мысль каждого слайда считывается за три секунды?
- У дека есть понятная narrative arc?
- Переходы между секциями очевидны?

### 2. Попадание в аудиторию

- Тон соответствует аудитории?
- Термины и предпосылки понятны?
- Плотность подходит ситуации?

### 3. Визуальная иерархия

- На каждом слайде есть ясная точка входа?
- Title, body, chart и caption достаточно различаются?
- Воздух направляет взгляд?

### 4. Качество исполнения

- Поля, сетка и интервалы едины?
- Похожие элементы выровнены одинаково?
- Цвета контролируются?
- Размеры и насыщенности шрифтов purposeful?

### 5. Готовность к экспорту

- HTML рендерится без ошибок?
- PDF/PNG совпадают с HTML?
- PPTX остается редактируемым, если он нужен?
- SVG-обещания ограничены constrained vector slides?

## Частые проблемы

1. Слишком много мыслей на одном слайде.
2. Заголовок повторяет график вместо вывода.
3. Body text слишком мелкий для выступления.
4. Все слайды используют один layout.
5. У графиков нечитаемые подписи.
6. Декор спорит с сообщением.
7. Визуальный язык случайно меняется между секциями.
8. Placeholder data не помечены.
9. У важных claims нет source notes.
10. Экспортные ограничения вспомнили слишком поздно.

## Шаблон отчета

```markdown
## Ревью дека

Общая оценка: X/10

Сохранить:
- ...

Исправить сначала:
- ...
- ...

Риск экспорта:
- ...

Быстрые улучшения:
- ...
```

Держите отчет коротким и применимым. Пользователю нужны решения и правки, а не
длинное эссе.
