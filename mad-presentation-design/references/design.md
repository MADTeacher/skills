# Дизайн презентации

Этот справочник задает базовые правила композиции. Он помогает быстро принять
решения: что должно быть на слайде, как зритель будет читать материал и где
нужны доказательства, а не украшения.

Подробные правила лежат отдельно:

- брендовые материалы и `brand-spec.md`: `brand-assets.md`;
- выбор визуального направления: `design-directions.md`;
- ограничения экспорта HTML/PPTX: `html-decks.md`, `pptx-authoring.md`.

## Быстрый порядок

1. Сначала выясните контекст: аудитория, цель, сюжет, формат сдачи, похожие
   презентации, бренд, данные, скриншоты, графики.
2. Если есть конкретный бренд или продукт, пройдите строгую проверку материалов
   из `brand-assets.md`.
3. Если стиль не задан, предложите 3 разных направления из
   `design-directions.md`.
4. Соберите проверку визуальной грамматики из 2 слайдов: обложка (`cover`) и
   плотный содержательный слайд (`content slide`).
5. После выбора зафиксируйте дизайн-токены (`design tokens`): цвет, шкала шрифтов, сетка,
   обработка изображений, стиль графиков, при необходимости шкала emoji, футер,
   нумерация и заметки. Дизайн-токены - это список постоянных настроек, чтобы
   все слайды выглядели как одна система.

## Правила композиции

- Один слайд выполняет одну работу: ориентирует, доказывает, сравнивает,
  объясняет процесс, просит решение или фиксирует следующий шаг.
- Если на слайде есть доказательство или график, заголовок должен быть выводом,
  а не названием темы.
- Доказательство важнее декора: реальный скриншот, график, диаграмма, цитата,
  изображение продукта или примечание об источнике.
- Компоновка должна поддерживать сюжет. Если меняется только цвет, направление
  еще не выбрано.
- Похожие элементы должны иметь одинаковые размеры, отступы и выравнивание.
- Футер, номер страницы и примечания об источниках должны быть стабильными на
  всех слайдах.

## Роли слайдов

Чередуйте роли, чтобы презентация не превратилась в один шаблон с новым
текстом:

1. Обложка (`cover`): задает тон, обещание и главный визуальный сигнал.
2. Повестка (`agenda`): показывает карту истории, а не обычный список пунктов.
3. Разделитель (`section divider`): дает ритм и отмечает смену темы.
4. Тезис (`claim`): одна мысль и одно доказательство или пример.
5. Вывод из данных (`data insight`): график/таблица + заголовок-вывод +
   источник.
6. Сравнение (`comparison`): варианты, сравнимые по одной логике.
7. Процесс (`process`): 3-6 шагов с параллельными подписями.
8. Цитата или голос клиента (`quote/customer voice`): крупная цитата с автором
   и контекстом.
9. Дорожная карта (`roadmap`): этапы, сроки, неопределенность и ответственные.
10. Финал и просьба (`closing/ask`): решение, ответственный, следующий шаг; не
    обычное "thank you".

## Читаемость

Для `1920x1080`:

- заголовок: обычно `56-96px`;
- заголовок раздела: `72-120px`;
- основной текст: `24px` или больше;
- подписи и примечания об источниках: обычно `18px` или больше;
- подписи на графиках должны читаться в режиме показа;
- обычный содержательный слайд (`content slide`): максимум 3-5 визуальных
  групп.

Для `960x540pt` редактируемого PPTX:

- сохраняйте иерархию в `pt`;
- не делайте тесные текстовые блоки;
- держите безопасные поля;
- сложные эффекты заменяйте простыми фигурами или слоями изображений, если
  текст должен остаться редактируемым.

## Учебная система эмоджи

На образовательных слайдах эмоджи можно использовать как учебные подсказки:
маркеры шагов, сигналы "пример / практика / проверка", мнемоники и
навигационные якоря. Эмодзи не заменяют диаграмму, график, скриншот,
предметное изображение, учебную иллюстрацию или доказательство.

Правила:

- Сначала определите аудиторию и тон: дети, подростки, взрослые,
  корпоративное обучение. Для взрослых используйте меньше эмоджи и более спокойную
  сетку.
- Для учебной темы сначала решите, нужен ли рисунок, фотография, карта, схема
  или другое предметное изображение. Если нужен, подберите материал и
  спланируйте подписи до выбора эмодзи.
- Задайте шкалу эмоджи в токенах, а не случайными inline-размерами.
- Используйте 3-5 повторяемых ролей подсказок: понятие, пример, упражнение,
  предупреждение, повторение. Одна роль = один эмоджи и один диапазон размеров.
- Меняйте размер осознанно: большой эмоджи для подсказки раздела или обложки,
  средний для карточки или шага, маленький для подписи в строке.
- Держите эмоджи в отдельной визуальной зоне: колонка маркеров,
  полоса упражнений или полоса повторения. Не вставляйте их хаотично в
  заголовки.
- Если эмоджи только украшает, добавляйте `aria-hidden="true"`. Если он несет
  смысл, повторяйте этот смысл текстом на слайде или в заметках.
- Для редактируемого PPTX не обещайте одинаковый цветной рендеринг эмоджи на
  всех системах. Проверяйте PNG/PPTX и при необходимости сохраняйте важный
  эмоджи как слой изображения.

Рекомендуемые CSS-токены для `1920x1080`:

```css
:root {
  --emoji-xs: 28px;
  --emoji-sm: 40px;
  --emoji-md: 64px;
  --emoji-lg: 96px;
  --emoji-xl: 136px;
}

.emoji {
  display: inline-flex;
  width: 1em;
  height: 1em;
  align-items: center;
  justify-content: center;
  font-family: "Apple Color Emoji", "Segoe UI Emoji", "Noto Color Emoji", sans-serif;
  line-height: 1;
}

.emoji[data-size="sm"] { font-size: var(--emoji-sm); }
.emoji[data-size="md"] { font-size: var(--emoji-md); }
.emoji[data-size="lg"] { font-size: var(--emoji-lg); }
```

## Анти-шаблонные правила

Избегайте:

- generic AI glow: типового "сияния про AI";
- purple-blue gradients и floating spheres: фиолетово-синих градиентов и
  парящих шаров;
- случайных emoji-icons вне учебной системы подсказок;
- фальшивых панелей показателей, фальшивых графиков и выдуманных чисел;
- угаданных логотипов, скриншотов и брендовых цветов;
- одинакового card stack на каждом слайде;
- силуэтов из стоковых шаблонов и абстрактных технологичных фонов без
  доказательств.

Используйте:

- реальные материалы и примечания об источниках;
- ясную типографическую иерархию;
- стабильную систему сетки и отступов;
- содержательные диаграммы и графики;
- честные заглушки с подписями;
- размерные эмоджи-подсказки для образовательных презентаций, когда они помогают
  учиться;
- одну сильную визуальную идею на ключевом слайде.

## Данные и утверждения

- Не выдумывайте цифры, ссылки на источники, цитаты и статус релиза.
- Помечайте демонстрационные данные прямо на слайде или в заметках.
- Таблицы используйте для точного поиска значений, графики - для закономерностей.
- Подсвечивайте одно число или одно изменение, которое аудитория должна
  запомнить.
- Источники держите рядом с графиком/таблицей или в заметках докладчика.

## CSS helpers

```css
* {
  box-sizing: border-box;
}

.slide {
  width: 1920px;
  height: 1080px;
  overflow: hidden;
}

.grid {
  display: grid;
  grid-template-columns: repeat(12, minmax(0, 1fr));
  gap: 24px;
}

p,
h1,
h2 {
  text-wrap: pretty;
}
```

Эффект должен создавать иерархию, доверие, ритм или ощущение материала. Если
после удаления эффекта смысл слайда не меняется, эффект лишний.
