# Безопасность SVG

Используйте этот файл при работе с чужим SVG, веб-встраиванием, внешними ссылками, ошибками рендера и финальной проверкой.

## Главный риск

SVG — это XML-документ с возможностью ссылок, стилей, анимации и встраивания. В небезопасной среде он может стать источником межсайтового скриптинга, загрузки внешних ресурсов или утечки данных.

## Запрещённые элементы и признаки

| Признак | Почему опасно | Что делать | Проверка |
|---|---|---|---|
| `<script>` | Запускает JavaScript | Удалить полностью | Скрипт не найден валидатором |
| `foreignObject` | Может включать HTML | Заменить SVG-формами | Элемент отсутствует |
| `iframe`, `object`, `embed` | Встраивает внешний контент | Удалить | Элементы отсутствуют |
| Атрибуты `onload`, `onclick`, `onerror` и похожие | Запускают события | Удалить все `on...` | Валидатор не видит обработчиков |
| `href="http..."`, `xlink:href="http..."` | Тянет сеть | Заменить внутренним `#id` или удалить | Остались только внутренние ссылки |
| `data:` | Может спрятать код или тяжёлый файл | Удалить или пересобрать как SVG-формы | `data:` отсутствует |
| `javascript:` | Прямая попытка выполнить код | Удалить | Подстрока отсутствует |
| `@import` в стиле | Загружает внешний CSS | Удалить | В стилях нет импорта |
| `DOCTYPE` и `ENTITY` | Риск внешних сущностей | Удалить | XML без сущностей |

## Разрешённые внутренние ссылки

Можно использовать ссылки вида `href="#symbolId"`, `fill="url(#gradientId)"`, `clip-path="url(#clipId)"`, `mask="url(#maskId)"`. Они должны ссылаться на элементы внутри того же SVG.

## Ручная проверка

- Найдите все `href`, `xlink:href`, `src`, `url(...)` и проверьте, что они не ведут в сеть.
- Найдите все атрибуты, начинающиеся с `on`, и удалите их.
- Найдите `script`, `foreignObject`, `iframe`, `object`, `embed`, `DOCTYPE`, `ENTITY`.
- Проверьте, что пользовательский текст экранирован: `&`, `<`, `>`, кавычки в атрибутах.
- Убедитесь, что итоговый SVG открывается как картинка и не требует внешних файлов.

## Командная проверка

Используйте Node.js-скрипт:

```bash
node .agents/skills/artistic-svg-director/scripts/svg_pipeline.mjs validate image.svg
node .agents/skills/artistic-svg-director/scripts/svg_pipeline.mjs audit image.svg preview.png
```

`validate` проверяет опасные конструкции. `audit` сначала проверяет SVG, затем пытается создать PNG для визуальной оценки.

## Сдача веб-файла

Если SVG пойдёт на сайт, отдавайте безопасный самодостаточный файл. Если нужна интерактивность, выделите её в отдельный кодовый слой и попросите пользователя подтвердить контекст публикации, политику безопасности сайта и права на внешние ресурсы.
