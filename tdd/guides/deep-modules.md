# Глубокие модули

## Определение

Глубокий модуль — это модуль с небольшим, стабильным публичным интерфейсом и существенной внутренней функциональностью.

```text
Маленький интерфейс
  ↓
Много полезной скрытой логики
  ↓
Устойчивое поведение
```

Мелкий модуль, наоборот, имеет большую публичную поверхность и мало собственной ответственности.

```text
Много методов и параметров
  ↓
Мало инкапсуляции
  ↓
Пользователь модуля вынужден знать внутренние детали
```

## Почему это важно для TDD

Глубокие модули проще тестировать через поведение.

Если интерфейс маленький и выражает доменную операцию, тесты могут проверять важные сценарии, не привязываясь к внутренней структуре.

## Пример глубокого интерфейса

```pseudocode
interface UserRegistration {
  register(input: RegisterUserInput): RegistrationResult
}
```

Внутри модуль может:

- валидировать email;
- нормализовать данные;
- проверять уникальность;
- создавать пользователя;
- сохранять запись;
- публиковать событие;
- возвращать результат.

Снаружи остается один понятный метод.

## Пример мелкого интерфейса

```pseudocode
interface UserRegistrationSteps {
  validateEmail(email): ValidationResult
  normalizeEmail(email): Email
  checkUniqueness(email): Boolean
  createUserRecord(email): UserRecord
  saveUser(record): SavedUser
  publishRegisteredEvent(user): Void
}
```

Такой интерфейс заставляет вызывающий код знать порядок шагов. Тесты начинают проверять оркестрацию вместо поведения.

## Как искать глубокий модуль

Перед реализацией спроси:

- Какая операция имеет смысл для пользователя или домена?
- Можно ли выразить ее одним публичным методом?
- Какие детали можно спрятать внутри?
- Какие параметры можно объединить в value object или input object?
- Какие ошибки можно вернуть как доменный результат?
- Будет ли интерфейс стабильным при смене внутренней реализации?

## Когда не нужно выделять модуль

Не выделяй модуль только ради слоя абстракции.

Модуль не нужен, если:

- он просто проксирует один вызов;
- его интерфейс повторяет внутреннюю зависимость;
- тесты через него не становятся понятнее;
- он появляется только для возможности замокать собственный код;
- он не скрывает значимой сложности.
