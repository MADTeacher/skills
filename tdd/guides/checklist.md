# Чеклист TDD-цикла

## Перед тестом

- [ ] Понятно, какое поведение нужно проверить.
- [ ] Поведение выражено через публичный интерфейс.
- [ ] Тест проверяет один логический сценарий.
- [ ] Тест не требует знания приватных методов.
- [ ] Выбран минимальный вертикальный срез.
- [ ] Понятно, почему тест должен упасть до реализации.

## RED

- [ ] Написан один тест.
- [ ] Тест падает.
- [ ] Тест падает по ожидаемой причине.
- [ ] Падение подтверждает отсутствие нужного поведения, а не ошибку теста.

## GREEN

- [ ] Реализация минимальна.
- [ ] Не добавлены будущие сценарии без тестов.
- [ ] Не замоканы внутренние модули собственного кода.
- [ ] Внешние зависимости подменены только там, где это оправдано.
- [ ] Новый тест проходит.
- [ ] Релевантные существующие тесты проходят.

## REFACTOR

- [ ] Рефакторинг начат только после GREEN.
- [ ] Изменения маленькие и обратимые.
- [ ] Поведение не изменено.
- [ ] Публичный интерфейс не усложнен без необходимости.
- [ ] Тесты по-прежнему проходят.

## Перед следующим срезом

- [ ] Текущее поведение зафиксировано.
- [ ] Следующий сценарий выбран осознанно.
- [ ] Нет незавершенного рефакторинга.
- [ ] Нет лишней реализации, для которой нет теста.
